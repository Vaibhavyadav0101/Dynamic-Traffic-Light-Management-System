// SYSTEM OVERVIEW //

HOW THE AI OPERATES

01
🗺️
SUMO SIM
Eclipse SUMO simulates real city traffic with vehicles, routes, and junctions loaded from network files.
02
👁️
STATE SENSING
TraCI API reads live vehicle counts per lane and waiting times at each of the 5 junctions every step.
03
🧠
DQN DECISION
Deep Q-Network evaluates state and selects the optimal traffic phase to minimise cumulative waiting time.
04
🚦
PHASE CONTROL
Selected phase is applied instantly via TraCI. Yellow transition + green phase durations are dynamically set.
05
📈
REWARD & LEARN
Negative waiting time becomes the reward signal. The agent updates weights via experience replay each step.
// TECHNICAL ARCHITECTURE //

SYSTEM COMPONENTS

NEURAL NETWORK
Deep Q-Network (DQN)
3-layer fully connected network: input(4) → FC(256) → FC(256) → output(4). Trained with Adam optimizer, MSE loss, γ=0.99. ε-greedy exploration decays from 1.0 → 0.05 over training.
STATE SPACE
Observation Vector
4-dimensional state per junction: vehicle count per lane (position > 10m from stop line). Updated every simulation step via traci.lane API calls.
ACTION SPACE
Phase Selection
4 discrete actions per junction — one per traffic phase. Each action triggers a yellow (6s) transition followed by a green (15s) phase. Phases auto-sized to match each junction's lane count.
REWARD FUNCTION
Waiting Time Penalty
Reward = −1 × total_waiting_time across all controlled lanes at each decision step. Encourages the agent to minimise vehicle halting across the entire network simultaneously.
BASELINE
Fixed-Time Controller
Cycles through 4 phases at fixed 30-step intervals regardless of traffic density. Metrics from train_fixed.py serve as the benchmark to evaluate RL improvement.
METRICS
AWT · ATT · AQL
AWT — Average Waiting Time per vehicle. ATT — Average Trip (Turnaround) Time. AQL — Average Queue Length per junction. All tracked per-junction and globally each epoch.
