// STEP BY STEP //

HOW THE AI OPERATES

The AI learns by doing — it runs thousands of simulations, makes mistakes, gets penalized, and gradually figures out the best traffic pattern on its own.

STEP 01
🗺️
LOAD MAP
SUMO loads the real road network with 5 junctions and vehicle routes from XML files.
? What is SUMO
Eclipse SUMO is an open-source traffic simulator. Think of it as a virtual city where we test our AI safely before real roads.
▶
STEP 02
👁️
SENSE STATE
Every step, the AI reads how many vehicles are waiting in each lane at every junction via TraCI.
? What is state
State = [vehicles in lane 1, lane 2, lane 3, lane 4]. This 4-number snapshot tells the AI how congested each direction is right now.
▶
STEP 03
🧠
DQN DECIDES
The neural network picks 1 of 4 traffic phases — whichever it predicts will reduce total waiting time most.
? What is DQN
Deep Q-Network: a neural network that maps states to Q-values (predicted future reward) for each action. Best Q-value wins.
▶
STEP 04
🚦
APPLY PHASE
The chosen phase is sent to SUMO: yellow transition (6s) then green (15s). Vehicles start moving.
? What is a phase
A phase is a specific green/red pattern. E.g. Phase 0 = North-South green. Phase 2 = East-West green. There are 4 per junction.
▶
STEP 05
📈
LEARN & IMPROVE
Waiting time becomes the penalty. The agent updates its weights so it makes better decisions next time.
? How does it learn
Reward = −waiting_time. The agent stores (state, action, reward, next_state) and trains its network via backpropagation every step.
// UNDER THE HOOD //

TECHNICAL ARCHITECTURE

Every component explained simply — what it does and why it matters.

NEURAL NETWORK
Deep Q-Network (DQN)
input(4) → FC(256,ReLU) → FC(256,ReLU) → output(4)

Adam optimizer · lr=0.1 · MSE loss · γ=0.99
Plain English: A small brain taking 4 lane-count numbers and outputting 4 scores — one per phase. Highest score wins.
EXPLORATION
ε-Greedy Policy
Starts at ε=1.0 (fully random), decays by 5e-4 per step to ε=0.05.
Plain English: Early on the agent tries random phases. Over time it trusts its own predictions more. By epoch 50 it's 95% using learned decisions.
REWARD SIGNAL
Negative Waiting Time
Reward = −1 × total_waiting_time across all controlled lanes per decision step.
Plain English: The AI gets "punished" more when vehicles wait longer, so it learns to keep traffic moving to avoid big punishments.
SIMULATION
Eclipse SUMO + TraCI
Maps: city1.net.xml + city1.rou.xml
TraCI: reads state, sets phases each step
Plain English: SUMO runs the virtual city. TraCI is the "remote control" our Python script uses to read sensors and push signal changes.
BASELINE
Fixed-Time Controller
Rotates 4 phases every 30 steps regardless of traffic. Runs via train_fixed.py.
Plain English: Like traditional traffic lights — they don't care if there are 50 cars or 0 cars. The RL agent is smarter because it adapts.
METRICS
AWT · ATT · AQL
AWT = avg seconds each vehicle waits
ATT = avg total trip duration
AQL = avg vehicles halted per junction
Plain English: Lower AWT = less waiting. Lower ATT = faster trips. Lower AQL = shorter queues. All 3 decrease as the AI trains.
